# World Cup 2026 产品规划

## 1. 产品主线

网站的核心不是浏览数据，而是完成一次有依据、可解释、愿意分享的预测。

主要用户路径：

1. 用户从「今日比赛」或「小组强度」发现感兴趣的球队。
2. 选择两支队伍，进入预测准备页。
3. 系统展示本次可用数据、更新时间和模型可信度。
4. 运算并动态展示胜、平、负概率以及关键指数。
5. 用户展开预测依据，调整个人偏好，生成可分享结果。

## 2. 移动端优先布局

### 全局导航

使用底部四栏导航：

- 今日
- 小组
- 预测
- 更多

「预测」是中间主操作，可使用更醒目的按钮样式。

### 今日页

- 顶部：日期、赛事状态、数据最近更新时间。
- 第一屏：横向滑动的精选卡片，包括今日最值得看、最大爆冷可能、强队稳胆。
- 第二屏：按时间排列的比赛卡片。
- 比赛卡片只展示最重要的信息：双方、开赛时间、胜率、观赛价值、爆冷指数。
- 点击卡片进入比赛详情；点击「预测」直接带入双方球队。

### 小组页

- 顶部横向滑动 Group A 至 Group L。
- 小组卡片展示死亡之组指数、球队强度分布和出线竞争程度。
- 点击小组后进入球队选择状态，依次选择两队并开始预测。

### 预测页

- 默认展示最近预测和推荐对阵。
- 球队选择器优先按小组筛选，也支持搜索。
- 选定两队后，底部固定主按钮显示「开始预测」。
- 结果页采用纵向故事流：
  1. 胜平负核心结果
  2. 模型可信度与数据新鲜度
  3. 决定结果的前三个因素
  4. 历史交锋时间线
  5. 详细模型依据
  6. 用户调整与分享

### 桌面端增强

桌面端沿用相同信息顺序，将移动端纵向内容扩展为双栏或三栏仪表盘，不另建一套产品逻辑。

## 3. 预测模型

### 推荐方法

使用可回测、可校准的混合模型，不直接手写固定百分比后长期不变。

基础模型输出：

- Elo 强弱概率
- 近期状态模型
- 历史交锋模型
- 进球分布模型（Poisson / Dixon-Coles）
- 市场赔率隐含概率
- 比赛上下文修正

再通过历史比赛回测，学习或校准各模型权重，输出最终胜、平、负概率。

### 历史交锋

历史交锋是重要信号，但应避免少量比赛影响过大。

- 时间衰减：建议使用指数衰减，半衰期约 2 至 3 年。
- 比赛重要性：世界杯与洲际大赛高于友谊赛。
- 场地修正：区分主场、客场、中立场。
- 样本收缩：交锋次数少时，将结果向双方整体实力预测收缩。
- 最大贡献：初期建议限制在最终模型的 10% 至 20%，通过回测再调整。

### 初始权重建议

无可靠赔率时：

- Elo 与长期实力：30%
- 近期状态：20%
- 历史交锋：15%
- 进球分布模型：25%
- 比赛上下文：10%

有可靠赔率时：

- 市场共识：25%
- Elo 与长期实力：25%
- 近期状态：15%
- 历史交锋：10%
- 进球分布模型：20%
- 比赛上下文：5%

这些权重只是初始值，正式版本必须通过时间切分回测和概率校准决定。

### 可信度

每次预测同时输出可信度，而不是只输出概率。

可信度取决于：

- 各子模型是否方向一致
- 可用历史比赛数量
- 数据新鲜度
- 阵容与伤停信息完整度
- 赔率是否可用且稳定
- 概率校准误差

## 4. 数据方案

### 本地历史数据

优先保存为本地数据库，建立统一球队 ID 和名称映射。

- `martj42/international_results`：国家队历史比赛、射手、点球大战，适合训练近期状态和历史交锋模型。
- `openfootball/worldcup.json`：历届世界杯与 2026 赛程，CC0，可作为赛程基础和校验源。
- `StatsBomb Open Data`：部分赛事的事件、阵容和比赛级数据，适合增强进球与比赛风格分析，但覆盖范围有限。

### 在线更新数据

- FIFA 官方页面：排名与赛程的最终校验源。
- API-FOOTBALL：赛程、比赛、阵容、伤停、统计、赔率和 head-to-head；适合作为主要实时数据候选。
- The Odds API：聚合市场赔率；适合作为赛前市场共识，但历史赔率成本较高。

### 数据管线

1. 原始数据写入 `raw` 层，保留来源、抓取时间和原始字段。
2. 标准化球队名称、比赛时间、比赛类型和场地属性。
3. 写入统一比赛表和球队表。
4. 定时生成模型特征，并保存特征版本。
5. 每次预测保存模型版本、数据快照、结果和解释。

避免直接依赖网页爬取作为唯一来源。公开 API 不足时才增加爬取，并为爬取源准备失败降级和人工校验。

## 5. UI 与动态表达

- 视觉主题可以采用「赛事数据控制台 + 球场能量」风格，避免传统新闻门户布局。
- 核心概率使用可动画过渡的分段概率条，保证数字始终清晰。
- 模型运算过程用 1 至 2 秒的阶段动画呈现：读取历史交锋、评估近期状态、校准结果。
- 历史交锋使用可滑动时间线，直观看到时间越近权重越高。
- 指数变化使用小型趋势图和轻量数字滚动，不使用持续无意义动画。
- 支持 `prefers-reduced-motion`，弱网与低性能设备减少动画。
- shadcn/ui 优先用于 Card、Drawer、Tabs、Chart、Progress、Slider、Badge、Skeleton 和底部操作区域。

## 6. 建议的 MVP 范围

第一阶段只做：

- 今日比赛展示
- 小组强度展示
- 任意两支参赛队伍预测
- 可解释结果页
- 历史交锋时间衰减
- 本地历史数据导入与模型回测

第一阶段暂不做：

- 登录与社区
- 实时滚球预测
- 复杂球员级模型
- 用户公开排行榜

## 7. 额外建议

- 增加「模型过去表现」页面，公开 Brier Score、命中率和回测区间，建立真实信任。
- 每个数据模块显示来源和更新时间，并允许用户查看详细说明。
- 预测结果区分「模型认为」与「市场认为」，分歧本身就是有价值的内容。
- 对没有直接历史交锋的球队，展示相似对手表现，避免历史交锋模块空白。
- 增加小组赛情景模拟，展示某场结果如何改变出线概率；这是比单场预测更有世界杯特色的功能。
